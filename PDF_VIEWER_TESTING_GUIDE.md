# PDF 查看器测试指南

## 🧪 测试环境准备

### 前置要求
- Docker 已启动
- 应用已部署到 http://localhost:3000
- 多页 PDF 文件（用于测试）

### 启动应用
```bash
# 如果使用 Docker
docker-compose up -d

# 或本地开发
npm run dev
```

## 📋 测试场景

### 场景 1: 基础 PDF 加载
**目标**: 验证 PDF 能正确加载

**步骤**:
1. 打开应用 http://localhost:3000
2. 创建新项目
3. 上传多页 PDF 文件
4. 点击 PDF 卡片打开查看器

**预期结果**:
- ✅ PDF 正确显示
- ✅ 第一页可见
- ✅ 无错误消息

### 场景 2: 多页滚动（iOS Safari）
**目标**: 验证 iOS Safari 上能滚动查看所有页面

**步骤**:
1. 在 iPhone 上打开应用
2. 打开多页 PDF
3. 向下滑动查看下一页
4. 继续滑动直到最后一页

**预期结果**:
- ✅ 能向下滚动
- ✅ 所有页面都可见
- ✅ 滚动流畅（无卡顿）
- ✅ 页码显示正确

### 场景 3: 缩放功能
**目标**: 验证缩放控制正常工作

**步骤**:
1. 打开 PDF 查看器
2. 点击 "+" 按钮放大（最多 200%）
3. 点击 "-" 按钮缩小（最少 50%）
4. 验证页码显示

**预期结果**:
- ✅ 缩放按钮响应
- ✅ 缩放百分比更新
- ✅ PDF 内容缩放正确
- ✅ 缩放范围 50% - 200%

### 场景 4: 不同浏览器兼容性
**目标**: 验证在各种浏览器上正常工作

**测试浏览器**:
- [ ] iOS Safari
- [ ] Chrome (iOS)
- [ ] Firefox (iOS)
- [ ] Chrome (Desktop)
- [ ] Firefox (Desktop)
- [ ] Safari (macOS)
- [ ] Edge (Desktop)

**预期结果**:
- ✅ 所有浏览器都能显示 PDF
- ✅ 所有浏览器都支持滚动
- ✅ 缩放功能在所有浏览器上工作

### 场景 5: 大文件处理
**目标**: 验证大 PDF 文件的处理

**步骤**:
1. 上传 10+ MB 的 PDF 文件
2. 打开查看器
3. 滚动查看所有页面
4. 测试缩放功能

**预期结果**:
- ✅ 文件加载成功
- ✅ 无内存溢出
- ✅ 滚动流畅
- ✅ 缩放响应快速

### 场景 6: 错误处理
**目标**: 验证错误处理机制

**步骤**:
1. 尝试打开损坏的 PDF
2. 尝试打开无效的 URL
3. 检查错误消息

**预期结果**:
- ✅ 显示友好的错误消息
- ✅ 应用不崩溃
- ✅ 能返回项目列表
